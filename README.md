# AEM Local SDK — Docker Environment

Runs AEM as a Cloud Service SDK locally across three services:

```
Browser
  ├── :4502  →  aem-author     (Docker — eclipse-temurin Java)
  ├── :4503  →  aem-publish    (Docker — eclipse-temurin Java)
  └── :80    →  aem-dispatcher (Docker — managed by SDK's docker_run.sh)
                    └── proxies to host:4503 → aem-publish
```

> **Note:** Author and Publish are managed by `docker compose`.
> The Dispatcher is managed separately by the SDK's own `docker_run.sh` script
> and must be started with `./start-dispatcher.sh` on the host.

---

## Prerequisites

- Docker Desktop installed and running
- AEM SDK zip downloaded from [Adobe Software Distribution](https://experience.adobe.com/#/downloads)
  - Requires an Adobe ID with AEMaaCS entitlement
  - Filename example: `aem-sdk-2026.2.24464.20260214T050318Z-260100.zip`

---

## First-Time Setup

```bash
# 1. Place your SDK zip in this directory and make scripts executable
chmod +x aem-setup.sh start-author.sh start-publish.sh start-dispatcher.sh

# 2. Run setup — extracts JARs, creates folders, writes .env
./aem-setup.sh

# 3. Start Author and Publish via Docker Compose
docker compose up --build

# 4. Start Dispatcher separately on the host (in a new terminal)
./start-dispatcher.sh
```

AEM takes **2–5 minutes** to boot on first start (repository initialisation).
Watch progress with:
```bash
docker compose logs -f aem-author
docker compose logs -f aem-publish
```

## Services

| Service | Port | Managed by | Purpose |
|---------|------|------------|---------|
| `aem-author` | 4502 | `docker compose` | AEM Author instance |
| `aem-publish` | 4503 | `docker compose` | AEM Publish instance |
| `aem-dispatcher` | 80 | `start-dispatcher.sh` | Dispatcher (SDK docker_run.sh) |

> The Dispatcher uses Adobe's own Docker image pulled internally by `docker_run.sh`.
> It cannot be nested inside Compose as it manages its own container.
> Never add it as a Compose service — it will fail with a pull access denied error.

## Project Structure

```
aem-sandbox/
├── docker-compose.yml        # Manages aem-author and aem-publish only
├── .env.example              # All configurable variables (copy → .env)
├── aem-setup.sh              # One-time setup: extract SDK, create dirs, write .env
├── start-author.sh           # Entrypoint for the author container
├── start-publish.sh          # Entrypoint for the publish container
├── start-dispatcher.sh       # Runs docker_run.sh on the host (not inside Compose)
│
├── aem-sdk-*.zip             # ← Place your SDK zip here (not committed to git)
│
├── author/                   # Created by aem-setup.sh
│   ├── aem-sdk-quickstart-*.jar
│   └── crx-quickstart/       # AEM repository (created on first boot)
│
├── publish/                  # Created by aem-setup.sh
│   ├── aem-sdk-quickstart-*.jar
│   └── crx-quickstart/
│
└── dispatcher/               # Created by aem-setup.sh
    ├── docker_run.sh         # Adobe's launcher (extracted from SDK .sh)
    ├── src/                  # Dispatcher config files
    └── lib/                  # Dispatcher shared libraries
```

## Configuration

All settings live in `.env` (generated by `aem-setup.sh` from `.env.example`):

| Variable | Default | Description |
|----------|---------|-------------|
| `AEM_SDK_ZIP` | auto-detected | SDK zip filename |
| `JAVA_VERSION` | `21` | Java image tag (`11`, `17`, `21`) |
| `AUTHOR_PORT` | `4502` | Author HTTP port |
| `PUBLISH_PORT` | `4503` | Publish HTTP port |
| `DISPATCHER_PORT` | `80` | Dispatcher HTTP port |
| `AUTHOR_JVM_OPTS` | `-Xmx2048m -Xms512m` | Author JVM heap |
| `PUBLISH_JVM_OPTS` | `-Xmx2048m -Xms512m` | Publish JVM heap |
| `AUTHOR_EXTRA_RUNMODES` | `nosamplecontent` | Extra run modes appended to `author` |
| `PUBLISH_EXTRA_RUNMODES` | `nosamplecontent` | Extra run modes appended to `publish` |
| `AEM_ADMIN_PASSWORD` | `admin` | Admin password for both instances |

After editing `.env`, restart the affected container:
```bash
docker compose up --build aem-author    # rebuild + restart author only
```

---

## Changing Java Version

Edit `.env`:
```env
JAVA_VERSION=21
```
Then rebuild:
```bash
docker compose up --build
```

> **Note:** AEMaaCS SDK is certified for Java 11 and 17. Java 21 may work
> but is not officially supported. Stick to Java 21 for production parity.

## Useful Commands

```bash
# Start Author + Publish
docker compose up --build

# Start Dispatcher (separate terminal — runs on host)
./start-dispatcher.sh

# View logs
docker compose logs -f
docker compose logs -f aem-author

# Stop Author + Publish
docker compose down

# Stop Dispatcher
docker stop aem-dispatcher   # container name used internally by docker_run.sh

# Full reset (wipes repository data)
rm -rf author/crx-quickstart publish/crx-quickstart
docker compose down

# Re-run setup with a new SDK version
rm -rf sdk-extracted/ author/aem-sdk-quickstart-*.jar publish/aem-sdk-quickstart-*.jar
rm -rf author/crx-quickstart publish/crx-quickstart
./aem-setup.sh
docker compose up --build

# Open a shell inside a running container
docker exec -it aem-author bash
```

## Updating to a New SDK Version

```bash
# 1. Place new SDK zip in this directory
# 2. Remove old extracted data
rm -rf sdk-extracted/ author/aem-sdk-quickstart-*.jar publish/aem-sdk-quickstart-*.jar
rm -rf author/crx-quickstart publish/crx-quickstart

# 3. Re-run setup (auto-detects newest zip)
./aem-setup.sh

# 4. Rebuild and start
docker compose up --build
./start-dispatcher.sh
```

## Troubleshooting

**`pull access denied for adobe/aem-ethos/dispatcher-publish`**
The Adobe dispatcher image requires credentials and cannot be pulled directly.
This means the dispatcher has been added to `docker-compose.yml` by mistake.
Remove it — the dispatcher must always be started via `./start-dispatcher.sh`
on the host, which delegates to the SDK's `docker_run.sh`.

**`aem-author` or `aem-publish` is unhealthy**
AEM first boot can take 3–5 minutes. The healthcheck has a 120s `start_period`
and 20 retries (10 minutes total). If it still fails:
```bash
docker compose logs aem-author | grep -i "error\|exception\|started"
```

**Out of memory / container killed**
Increase heap in `.env`:
```env
AUTHOR_JVM_OPTS=-Xmx4096m -Xms1024m
```
Also ensure Docker Desktop has at least **8 GB RAM** allocated
(Docker Desktop → Settings → Resources).

**Dispatcher `docker_run.sh` not found**
Check that `aem-setup.sh` successfully extracted the dispatcher tools:
```bash
ls dispatcher/
# Should contain docker_run.sh, src/, lib/
```
If empty, re-run `./aem-setup.sh` and check for warnings about the dispatcher `.sh` extractor.

**Port already in use**
Change ports in `.env` and restart:
```env
AUTHOR_PORT=14502
PUBLISH_PORT=14503
```

## .gitignore

Rename `gitignore.txt` to `.gitignore` after cloning. Key ignores:
- `aem-sdk-*.zip` — large binary, download manually
- `author/crx-quickstart/` and `publish/crx-quickstart/` — generated, can be GBs
- `.env` — contains credentials
- `sdk-extracted/` — regenerated by `aem-setup.sh`
